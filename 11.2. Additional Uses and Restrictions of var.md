##### Quick Question
<span style="color: #F0F8FF;">What happens?</span>
```Java
class X {
	var y = 99;
}
```
Answer: This fails, `var` is not permitted on fields

<span style="color: #F0F8FF;">What happens?</span>
```Java
for (var var = 0; var < 10; var++) {
	System.out.println(var);
}
```
Answer: This succeeds

<span style="color: #F0F8FF;">What happens?</span>
```Java
try (var in = Files.newBufferedReader(Path.of("data.txt"))) {

} catch (var e) {

}
```
Answer: resource declaration is good

But: `catch` fails-the exception type is not explicit

<span style="color: #F0F8FF;">What happens?</span>
```Java
public void doStuff(var x) {
	System.out.println(x);	
}
```
Answer: `var` cannot be used for normal method parameters

#### Complex Inferencing
Remember that the conditional operator and case expression can create expressions that have a type involving multiple interfaces
- `var` can express this
- resulting variable might have a type that cannot be expressed explicitly

```Java
package nondenotable

public class VarExamples {
	public static void main(String[] args) {
		String s = "Hello";
		Integer i = 99;
		var res = true ? s : i;
		res.compareTo(null);
		
		int x = 99;
		var res2 = switch(x) {
			case 99 -> s;
			default -> i;
		};
		
	}
}
```

##### Question Deep Dive
Which are valid individually?
```Java
A) public var x = 99;
B) var var;
C) var ia = new int[10];
D) var [] ia = new int[]{};
E) var a = 99, b = 100;
F) try (var in = new FileReader("x");
			var out = new FileWriter("y");)
G) catch (SQLException e) {var ex = e;}
```
Answer:
A fails since var cannot be used in a field
B fails it is not initialized at all
C succeeds
D fails we cannot use square bracket `[]` with var
E fails we can only use `var` to declare a single variable
F succeeds
G succeeds
