#### Initializing Static Fields
A static field is initialized during classloading
- To zero / zero-like values
- Using an assignment from an expression in the declaration
- In any number of static initializer blocks
Notes
- Initializations execute in order from top to bottom
- Must declare before making an unqualified read
- Only an initializer block can express flow control
- No checked exceptions can be unhandled

##### What happens?
```Java
class X {
	static BufferedRaeder fr = new BufferedReader(new FileReader("config.data"));
	
	static {
		confText = fr.readLine();
	}
	static String confText;
}
```
Answer: Compilation fails, both `new FileReader` and `fr.readLine()` throw checked exceptions
`======`

What happens?
```Java
class X {
	static int x;
	static {
		System.out.println("x is " + x);
		System.out.println("text is " + text);
	}
	static String text;
}
```
Answer: Compilation fails. `text` is referred to, unqualified, before declaration (X.text is ok)

`======`
How can this be corrected?
```Java
class Storage {
	static Driver driver = DriverManager.getDriver("jdbc:mydb:table");
}
```
Answer: Use a static initializer
```Java
class Storage {
	static Driver driver;
	static {
		try {
			driver = DriverManager.getDriver("jdbc:mydb:table");
		} catch (SQLException e) {
			// recover or wrap as runtime
		}
	}
}
```

`======`
```Java
class TryIt {
	public static void main(String[] args) {
		out.println(x + ", " + y + ", " + z);
	}
	static int x;
	static {out.println(x + ", " + TryIt.y);}
	static int y = 100;
	static {out.println(x + ", " + TryIt.y);}
	static {x = y + 10;}
	static int z = -1;
}
```
Answer: 0, 0
0, 100
110, 100, -1
