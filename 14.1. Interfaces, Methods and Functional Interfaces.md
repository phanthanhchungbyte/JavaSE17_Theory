#### Interface Method Types
An interface can declare several method types
- Abstract methods, without a body, ending with semicolon
	- which are public, instance, methods
- Static methods
	- which may be public or private
	- and *cannot* be invoked using an instance prefix
- Private methods
	- which may be instance or static
- Default methods
	- which are public, instance, methods, and have a body

#### The Receiver Parameter
In an interface, an instance method with a body
- must be either default or private
- has a receiver parameter `--this--` of the interface type
The receiver parameter may optionally be declared explicitly, as with class instance methods, like this:
```Java
interface A {
	default void doA(A this) {}
	private void doThat(A this)
}
```

#### Why two kinds (default vs private)?
 **`default` methods** - The Public Evolution Tool

- **Purpose**: Add new methods to interfaces without breaking existing implementations
    
- **Accessibility**: `public` (implicitly or explicitly)
    
- **Inheritance**: Can be overridden by implementing classes
    
- **Use case**: Library evolution (adding `stream()` to `Collection`)
    

 **`private` methods** - The Code Sharing Tool (Java 9+)

- **Purpose**: Share code between `default` methods
    
- **Accessibility**: Only within the interface
    
- **Inheritance**: Cannot be overridden or accessed by implementing classes
    
- **Use case**: DRY (Don't Repeat Yourself) in interfaces

#### More Interface Rules
Methods in an interface must not be final
Features of interfaces are public by default
Methods without a body are public and abstract by default
Nothing in an interface can have protected or package access.
- default access is public
Interfaces can declare public static final fields
- These are public, static and final even if this is not specified
- These can be of mutable object types
Interfaces cannot have initializers or static initializers

##### Quick Question
What do you notice?
```java
interface A {
	public void m1();
	protected void m2();
	void m3();
	private void m4();
	
	public void m5() {};
	protected void m6() {};
	void m7() {};
	private void m8() {}
}
```
Answer:
- m1 and m3 are valid public abstract method declarations
- m8 is a valid private instance method declaration
- Nothing can be protected in an interface
- Methods with no explicit accessibility are public
- A private abstract method (m4) can't be implemented, and is invalid
- m7 would be valid if marked default

#### Interfaces Extending Interfaces
An interface can declare that it `extends` another interface
- abstract and default methods of the parent interface will be part of the child interface
- private and static methods are not inherited
- the child interface can redeclare a parent method
	- abstract can be redeclared as default
	- default can be redeclared as abstract in which case a child of this cannot use the parent's default method and must declare its own implementation

##### Quick Question
What happens?
```Java
interface A {
	default void doStuff(int x) {}
}

interface B extends A {
	void doStuff(int y);
}

class X implements A {}
class Y implements B {}
```
Answer: Compilation fails at `class Y` because it does not implement the abstract method `doStuff` from `interface B`

#### Functional Interfaces
A *functional interface* is one that declares *exactly one* abstract method
- whether directly or by extending another interface
- excluding methods that are part of `java.lang.Object`
The annotation `@FunctionalInterface` (in the package `java.lang`) servers to document the intention of providing exactly one abstract method, and to request the compiler issue an error if this is not achieved.

A lambda expression can implement a functional interface

##### Quick Question
Which can be annotated `@FuntionalInterface`?
```java
interface A {
	boolean equals (Object o);
}
interface B {
	static void m1() {}
	void m2();
}
interface C extends B {
	default void m2() {}
}
interface D {
	void m3();
}
interface E extends D {}
```
Answer:
A cannot since boolean equals that takes Object method, maps to Object class's equals method (violation)
B is acceptable: - static method is not considered
                  - void m2() is abstract method so ok
C fails since default method is not abstract (has an implementation)
D is valid since it has exactly 1 abstract method
E is valid since it extends D and doesn't add any more abstract methods (only 1)

#### Fields in Interface
An interface has very limited ability to declare fields
- which must be public, static and final
- and will be so by default
- specifying any contradictory modifiers causes compilation to fail
Default the requirement for these fields to be static and final, they are not restricted to immutable types

##### Quick Question
Which are legal in an interface?
```Java
int a = 99;
public int b = 100;
final int c;
private static final int d = 102;
public static final StringBuilder e = null;
```
Answer: a, b and e
- a is implicitly public static final (acceptable)
- b using keyword public (redundant but acceptable)
- c is uninitialized and it is a final field (fails)
- d is private static final (not permitted)
- e is redeclaring the implicit access modifier (redundant but acceptable)
