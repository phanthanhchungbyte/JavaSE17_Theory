#### Reachability and Garbage Collection
If a program could possibly use an object then that object will not be garbage collected
- At intervals, the JVM determines all the objects that are reachable
	- this means all objects that could possibly be used
	- i.e., some variable or expression can get to the object
- *Unreachable* objects are then *eligible* for garbage collection
	- Whether or not their memory is *actually recovered* for reuse is not usually important to the program itself*
- Different garbage collectors implement *collection* differently, but *reachability* is independent of GC

#### Roots of the Search
Reachability analysis is transitive and starts from
- The entire live stack of every live thread
- All static fields of all loaded classes

Reachability is not "reference counting"

#### How References Disappear
Understanding how an object becomes unreachable is crucial

References cease to refer to a particular object when
- reassigned to refer to a different object, or
- reassigned to `null`
Local references go out of scope when their method completes
- even without reassignment, these no longer refer to anything

##### Quick Question
How many objects are unreachable just prior to this method returning?
```Java
void doStuff(LocalDate l) {
	LocalDate l1 = LocalDate.now();
	LocalDate l2 = LocalDate.now();
	l = null;
	l2 = l;
}
```
Answer: One
