#### The Queue Interface
The `Queue` interface is a sub-interface of `Collection`
A `Queue` accepts items at one end, and gives access to them, in the order accepted, at the other end.
- Items can be removed or examined in place
- Iteration takes items in the order they were added
- Choose an exception or a sentinel value on failure

|         | Exception on failure | Sentinel value on failure |
| ------- | -------------------- | ------------------------- |
| Insert  | `add`                | `offer`                   |
| Remove  | `remove`             | `poll`                    |
| Examine | `element`            | `peek`                    |

The `Deque` interface is a sub-interface of `Queue`
A `Deque` provides insertion, removal, and examination at either end of the structure

#### Using Deque as a Stack
The `Deque` interface also provides stack methods, `push`, `pop`, `peek`

These correspond to `addFirst`, `removeFirst`, and `getFirst`

`Deque` also provides additional methods:
- `removeFirstOccurrence` and `removeLastOccurrence` which remove an element by `Object` match
- `descendingIterator`

#### Equality in Queue and Deque
Implementations of `Queue` and `Deque` are *not* expected to provide meaningful equals/hashcode methods

##### Quick Question
What happens?
```java
Deque<Integer> di = new ArrayDeque<>();
di.addFirst(1); di.addFirst(2);
di.addLast (3);
System.out.print(di.removeFirst() + ", ");
System.out.print(di.removeLast() + ", ");
System.out.print(di.getLast() + ", ");
System.out.print(di.removeFirst() + ", ");
System.out.print(di.removeLast() + ", ");
```
Answer: `2, 3, 1, 1,` followed by an exception

#### The `Map` Interface
A map structure provides access to data values that are indexed through keys that are unique and organized for fast lookup. Java's `Map` interface defines the API for such a structure.
Many operations return the value that was in the map prior to the method call. 

#### Key Methods of the `Map` Interface
`put(key, value), putIfAbsent(key, value)`
`putAll(otherMap)`
`get(key), getOrDefault(key, default)`
`remove(key), remove (key, value)`
`replace(key, value), replace(key, old, new)`
`containsKey(key), containsValue(value)`
`clear()`
`size(), isEmpty()`

Obtain iterables:
`entrySet(), keySet(), values()`

Factories: `Map. of (k1, v1,...), Map. copyOf(map)`

A `Map. Entry` is a key-value pair, or "tuple"
`Map.entry(k, v)`

Factory: `Map. of Entries (entry...)`

#### Functional Methods of Map
`forEach (biConsumer)`
`merge(key, value, remap)`
1. finds the value associated with the specified key, or null
2. if null, inserts key - value as a new pair
3. if not null, calls remap with the provided value and found
value
4. if the result of the remap is non-null, replaces the existing
value with that result
5. if the result is null, remove the existing mapping

`compute (key, remap)`
`computeIfPresent (key, remap)`
`computeIfAbsent (key, remap)`
1. finds the value associated with the specified key, or null
2. IfPresent/IfAbsent versions quit if null/not-null
3. calls remap with the provided key and found value
4. if the result of the remap is non-null, insert a new pair, or
replace the existing value with the result
5. if the result is null, remove the existing mapping

##### Quick Question
What happens?
```Java
LocalDate today = LocalDate.now();
Map <LocalDate, String> mls =
	new HashMap<> (Map.of(today, "leave early"));
mls.merge(today, "go to party", (S1, S2) -> s1 + " and then " + S2);
System.out.println(mls);
```

Answer: Prints a map containing a key of today's date,
and a value of "leave early and then go to party"

What happens?
```java
LocalDate today = LocalDate.now();
Map<LocalDate, String> mls = Map. of (today, "leave early");
mls.compute(today, (d, s) -> null);
System.out.println(mls);
```

Answer: Throws an `UnsupportedOperationException`,
the map returned by `Map. of` is unmodifiable

##### Question Deep Dive
Which describe the result?
```java
var sa = new String[] {"one", "two", "three"};
var ls = Arrays.asList(sa);
var ss = Set.of ("un", "deux", "trois");
sa[0] = "zero";
int i = 0;
for (String s : ss) {
	System.out.printf("%7s : %s\n", ls.get(i++), s);
}
```

A) Compilation fails or an exception occurs
B) Left column lists `one two three`
C) Left column lists `zero two three`
D) Left column lists `one two three` in unpredictable order
E) Right column lists `un deux trois`
F) Right column lists `trois deux un`
G) Right column lists `un deux trois` in unpredictable order

Answer: Option C and G is correct
