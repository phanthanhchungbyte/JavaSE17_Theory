#### Terminal Operations of Streams
Any operation on a `Stream` that does not return a `Stream` object ends the pipeline and is called a *terminal operation*

There are several terminal operations in the core API, several of which exhibit laziness

How many elements must we examine to determine if all elements match a predicate?
- As soon as any one fails, the answer must be false, there's no need to look any further
- But we must look at them all to determine a true result

#### Primitive Stream Terminal Methods
Primitive streams provide additional terminal methods related
to the numeric nature of the data
`average, max, min, sum, summaryStatistics`

#### Examining Laziness
The `stream` API seeks to be lazy where possible
- Assembling a pipeline with `map`, `filter`, and `flatMap` does not process data
- Terminal operations "pull" data through the pipe
- Some built-in terminal operations quit pulling from the stream as soon as they know the result

Special case optimizations might exist that avoid processing a pipeline entirely.

#### Handling Infinite Streams
Streams that create data by computation, or are supplied from a source such as a network, can be infinite / unbounded
- but, generally, a result is only available when the stream is exhausted

This can be handled two ways
- complete-fast / lazy terminal operations
- truncating the stream
	- `limit(count)`
	- `takeWhile(predicate)`

#### The Optional Class
If we search for the maximum value of an empty stream, what should be the result?
- other situations might similarly have no meaningful result

Representing "there isn't one" using null is error prone; the stream APIs use the `Optional` class instead
This is a monad-like wrapper that contains one or zero items
- with `map, flatMap, filter`
- `ifPresent` takes the place of `forEach`
If a mapping operation produces `null`, the resulting optional will be empty

##### Quick Question
What happens?
```java
Stream.generate(() -> Math.random())
	.mapToInt(x → (int)(4 * ×))
	.filter(x -> x > 0)
	.allMatch(x -> x % 2 == 0);
```
Answer: the expression evaluates to `false`
The generate method creates random numbers, boxed as Double objects, in
the range 0 <= value < 1. These are converted to primitive int values in the
range 0 <= value <= 3, which are filtered to remove the zero values. Finally,
these are tested to see if all the values are even. Since they are not, the
stream evaluation terminates at the first odd value, and the stream expression
evaluates to false.

What happens?
```java
new Random().doubles(-1.0, +1.0)
	.average()
	.ifPresent (System.out::println);
```
Answer: the program hangs.
The `doubles` method creates an infinite stream, and with neither a limit, nor a short-circuiting terminal operation, it runs forever.
