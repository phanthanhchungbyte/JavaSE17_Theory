#### What is Scope?
The scope of a declaration  is the region where the declared entity can be accessed using a simple name, provided it's not shadowed
- A *simple name* means no preceding "something-dot"
e.g.:**balance**, and **name** 
but not: `account.balance`, `user.name`
(although `account` and `user` are simple names in this case)

#### Scope and Accessibility
Scope determines if a simple name is sufficient to reach the item

Accessibility determines if access will be granted, regardless of whether a simple or qualified name was used to reach it

Scope and accessibility are independent of one another
- Often, a variable that's in scope will be accessible, but not always

#### Kinds of Declaration Related to Scope
There are several kinds of declaration, and these govern scope

|      **Declaration kind** | **Scope**                                                                                     |
| ------------------------: | --------------------------------------------------------------------------------------------- |
|                     Local | from the point of declaration to end of immediately <br>enclosing block                       |
|                    Member | throughout the enclosing type, all nested types, and <br>subtypes to which it's inherited     |
| Formal <br>parameter-like | from the point of declaration to the end of the<br>following block, statement, or lambda body |
|           Static imported | all types in the compilation unit (source file)                                               |

#### Formal Parameter-Like Declarations
These are found in:
- Method, constructor, and lambda formal parameters
- Variables declared in a `for` loop of either kind
- Exceptions declared in a catch clause
- Resources in a `try`-with-resources construct

#### Redeclaration of an Identifier
In general*, declaring a second item with the same name as one already in scope will be rejected if there would be no way to qualify the names so as to disambiguate them

For example, we cannot declare:
- Two local variables in the same block with the same name
- Two fields in the same class with the same name
But nested class boundaries create a new scope and generally provide a means of disambiguation

#### Scope of enum Values in a switch
Where a switch is performed on an enum type
- the enum constants are in scope
- the qualified form of the constants must not be used
```Java
Days today = Days.MONDAY;
switch (today) {
	case MONDAY:
		System.out.println("Let's get to it");
	case Days.TUESDAY: // Not allowed
}
```

#### An import Brings Items Into Scope
The `import` statement brings types into scope (allowing them to be used without qualification)
- An `import static`  statement can bring `static` types, methods and fields (constant or otherwise) into scope
