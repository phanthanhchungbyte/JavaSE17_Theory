#### Review of Reference Casting Rules
A cast *down* the *class* hierarchy succeeds at compilation time
If the object not of the cast type, the cast fails at runtime
- `ClassCastException` is thrown
Casts *up* the *class* hierarchy are always valid, but redundant
- The compiler permits them (your IDE might complain)
A cast *across* the *class* hierarchy is rejected at compile time
- The two classes do not share any "is-a" relationship

But what about `interfaces`?

#### Reference Casting Rules
Generally if `A a = b`; is *possibly valid*, then `(B)a` is compilable
- It might fail at runtime
If `A a = b`; is definitely invalid, then `(B)a` will not compile
How do casts with interfaces behave?

```Java
import java.util.ArrayList;  
  
public class PossibleCasts {  
  public static void main(String[] args) {  
    ArrayList a1 = null;  
    Runnable rn = (Runnable) a1;  
    Runnable r = null;  
    ArrayList a12 = (ArrayList) a1;  
  
    // Impossible  
//    String s1 = null;  
//    rn = (Runnable) s1;  
//    Runnable r2 = null;  
//    String s2 = (String) r2;  
  }  
}
```
##### Quick Question
From a perspective of casting, which is the odd one out?
```Java
interface A {}
class B implements A {}
class C {}
enum D {}
```
Answer: `enum D` because an `enum` is implicitly `final`
