#### Primitive Streams
Because boxing and unboxing primitives is inefficient when used
extensively, the stream API has three variations for handling
primitive data directly
- `IntStream`
- `LongStream`
- `DoubleStream`

Primitive and object streams provide means to convert between
these types
- `mapToInt, flatMapToInt`
- `mapToLong, flatMapToLong`
- `mapToDouble, flatMapToDouble`
Primitive streams also provide conversion to object streams
- `mapToObj, flatMapToObj`
- `boxed`

Answer:
`map(String::length)` causes boxing
`map(String::toUpperCase)` does not cause boxing
`mapToObj` might result in boxing
`mapToInt` might result in unboxing
`boxed` causes boxing
#### Creating Streams
Several factory methods for creating streams are defined in each of the base interfaces
- `Stream.empty` -- makes a stream of zero items
- `Stream.of` -- makes a stream from the argument items
- `Stream.ofNullable` -- makes a stream of zero or one item
- `Stream.concat` -- joins two streams, maintaining laziness
- `Stream.generate` -- repeatedly calls a Supplier
- `Stream.iterate` -- takes a starting value, and a `UnaryOperator` that creates each subsequent value by processing the previous

`Stream` also provides a builder
- The `Stream.builder` method creates the builder object
- The builder object provides an add method, and implements `Consumer` to provide an alternate way to add items
- Building is a two-part process
	- assemble all the items
	- call the `build` method of the builder, which returns the stream in a ready-to-go state

#### Additional Stream Sources
Some other API features can create streams

- `Arrays.stream` variants
- `Collection.stream, Collection.parallelStream`
- `StreamSupport.stream (Spliterator s)`
- `Pattern.splitAsStream`
- In class `Random`, overloads of `doubles`, `ints`, and `longs`
- In the `Files` class
	- `find/walk, list/newDirectoryStream`
	- `lines`
#### Creating Integer Streams
`IntStream` and `LongStream` define two additional factories

These both create a monotonically increasing series starting with the first argument value
- `range` -- ends before the second argument value
- `rangeClosed` -- ends after emitting the second argument value

##### Quick Question
How many ways can you think of to create a stream of zero elements?

Answers:
```java
Stream.empty()
Stream.of()
Stream.ofNullable(null)
List.of().stream()
Stream.iterate(null, v -> v != null, v -> v)
Stream.builder().build()
IntStream.range(0, 0)
Arrays.stream(new Object[0])
new Random().doubles (0, 0.0, 1.0)
```

How do you process the key-value pairs of a map using a stream?

Answer:
extract the set of entries, then extract a stream from that set `myMap.entrySet().stream()`
This will create a `Stream<Map. Entry<K, V>>`

How can you create a stream of the numbers 1, 3, 5, 7 ... up to 1000

Answer:
```java
IntStream.iterate(1, v -> v < 1000, v -> v + 2)
IntStream.iterate(1, v -> v + 2).limit(500)
IntStream.rangeClosed(1, 999).filter(v -> v % 2 != 0)
IntStream.range(1, 1000).filter(v -> v % 2 != 0)
IntStream.range(0, 500).map(v -> v * 2 + 1)
```

#### Overview of `Spliterator`
The `Spliterator` exists to support parallel operations
- It provides access to separate "chunks" of data from some source
	- Lazy reading might still be possible, depending on source
	- Each chunk can then be handled in a separate thread
- Infrastructure calls the `trySplit` method as needed to create the separate "chunks"
	- Calls are made as needed to maintain parallelism
- A `Stream` can be built directly from a `Spliterator` using: `StreamSupport.stream(Spliterator s)`
