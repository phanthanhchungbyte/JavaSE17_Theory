#### Basics of Enumerated Types
Enumerated types
- Provides named instances, defined at compilation time
	- e.g., MONDAY, TUESDAY, etc.
- Values *can't* be added or removed dynamically
	- If the value set changes, recompile the program
- The instances are type-safe
	- unlike when using numeric values to indicate the values

Java provides these using a special class declared as `enum`

```Java
public enum Suit {
	HEART, DIAMOND, CLUB, SPADE;
}
```

#### Ensuring Fixed Value Count
To ensure that the only values that ever exist are those declared in the source
- enum types are implicitly final classes
- all constructors are implicitly private
	- attempting any other accessibility causes an error
- the type is protected against reflective access
- the type behaves properly when deserialized

Because all ways of obtaining an enum value produce the same objects, comparison with `==` is reliable

#### Initializing Enum Values
Enum values are initialized in the same way as regular classes with some variation
- The values are represented internally as public, static, final references to instances of the enum class
	- Because they have static final references, the instances must be constructed during *class loading*
	- so instance initializers and constructors run *before* static initializers!

#### Constructors for Enum Values
Enum types can have programmer-provided constructors
- If none are coded, a private default constructor is created
- All constructors must all be private
	- the default is private
	- attempts to select other accessibilities cause errors
- Overloaded constructors are permitted

```Java
enum DayOfWeek {
	SUNDAY, MONDAY("Back to work"), TUESDAY, WEDNESDAY(),
	THURSDAY, FRIDAY("Weekend tomorrow!"), SATURDAY;
	
	DayOfWeek() {} // allow "zero argument" construction
	DayOfWeek(String msg) {} // or with a String argument
}
```

##### Quick Question
What happens?
```java
enum DayOfWeek {
	SUNDAY(), MONDAY("Back to work"), TUESDAY, WEDNESDAY,
	THURSDAY, FRIDAY("Weekend tomorrow!"), SATURDAY;
	
	DayOfWeek(String msg) {}
	
	public static void main(String[] args) {
		System.out.println(MONDAY);
	}
}
```
Answer: Compilation fails because there is no zero-argument constructor

What happens?
```java
enum DayOfWeek {
	SUNDAY, MONDAY, TUESDAY, WEDNESDAY,THURSDAY, FRIDAY, SATURDAY;
	public DayOfWeek() {}
	
	public static void main(String[] args) {
		System.out.println(MONDAY);
	}
}
```
Answer: Compilation fails because the constructor has the modifier `public`

What happens?
```java
enum DayOfWeek {
	SUNDAY, MONDAY, TUESDAY, WEDNESDAY,THURSDAY, FRIDAY, SATURDAY
	DayOfWeek() {}
	
	public static void main(String[] args) {
		System.out.println(MONDAY);
	}
}
```
Answer: Compilation fails because there is no semicolon after the list of values
