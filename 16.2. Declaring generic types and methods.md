#### Multiple Generic Type Parameters
One type can have multiple type parameters, in a comma-separated list
- Consider the `java.util.Map` interface
```java
Module java.base
Package java.util

Interface Map<K,V>
```
Type parameters: 
K - the type of keys maintained by this map
V - the type of mapped values

#### Generic Method Declaration
Methods can use generics of their own
- Static methods can only use their own type parameters, but instance methods can use those of the class too
Declare type parameters for methods before the return type
```java
public static <K, V> void processElement(
	Map<K, V> map, K key, Consumer <V> op) {
	
	V element = map.get(key);
	if (element != null) op.accept(element);
}
```

Called like this:
```java
Map<Integer, String> mis = Map.of(1, "One", 2, "Two");
processElement(mis, 1, v -> System.out.println("Number is " + v));
```

#### Generic Method Declaration
Instance methods can add their own type parameters to those of the enclosing type
```java
class Container<E> {
	private List<E> items;
	public Container(List<E> items) {this.items = items;}
	public <F> Container <F> map(Function<E, F> op) {
		List<F> res = new ArrayList<>();
		for (E e : items) res.add(op.apply(e));
		return new Container<>(res);
	}
}
```

#### Generics in Sub-Typing
Generic types can be passed to a generic parent type
```java
class Pair<E> {E left, right;}

class TextPair<F extends CharSequence> extends Pair<F> {
	public boolean sameLength() {
		return left.length() == right.length();
	}
}
```
- The (bounded) declaration of F follows the `TextPair` type name
- `extends Pair<F>` is a *use* of that type F
