#### Ordering by Subtraction
Java's comparison methods produce a result that is analogous to
a subtraction operation
- 3-7 => -4, and 3 comes before 7
- 8-2 => +6, and 8 comes after 2
- 5-5 => 0, and 5 is at the same point in order as 5
*Always first argument minus second argument*

#### The Comparable Interface
`Comparable<E>` gives a class "natural order"
- implemented by the class having that order
- therefore only one natural order per class
- instance method `int compareTo(E other)`
- makes sense for many math concepts, e.g. Integer, but not so much for many business domain objects

#### The Comparator Interface
`Comparator<E>` defines an ordering for a type
- implemented by an independent class
- many comparators can exist for the same type
- instance method `int compare(E e1, E e2)`
- makes sense for business domain uses, e.g. order by height, expenditure, geographic distance from this office
	- note that these orders are not intrinsic features of the business domain class, they are ways of using it

#### Producing a Simple it value
It's tempting to perform a simple subtraction to create the
required `int` value, however
- `int` subtraction has the potential to underflow, producing
nonsense
- performing subtraction with long, float, or double, followed by casting can also fail due to over/underflow, or loss of precision with the floating point types
Use the primitive wrapper methods `compare` or `compareTo`

#### Comparing Text
`String, StringBuilder`, and `StringBuffer` all provide
"natural order" by implementing the `Comparable` interface

`String` (only) also provides
- `int compareToIgnoreCase(other)`
- `Comparator<String> CASE_INSENSITIVE_ORDER`

But all these comparisons are unaware of locale, `Collator` and `CollationKeys` provide for international use

##### Quick Question
If `a.compareTo(b)` returns `-1`, what do you know about the
relative older of a and b?

Answer: a comes *before* b

`-----------------------`
What interface defines the method `compareTo(E e)`? When
would you implement it?

Answer: `Comparable<E>`
Used to provide a class with a "natural" order,
such as the ordering of integers
