#### Levels of Accessibility for Attributes
Attributes (methods, fields, types) can have one of four modifiers indicating levels of accessibility
- `private`
	- accessible anywhere inside the top-level curly braces that surround the declaration
- default (no keyword used)
	- accessible anywhere in the same package
- protected
	- as default *plus* accessible in subclasses, provided the reference is of the subclass type

#### **Correct `protected` Rule:**

`protected` = **default (same package)** + **inheritance access in subclasses (even in different packages)**

But crucially:

- ✅ Accessible via **inheritance** (`this.member`, `super.member`, or just `member`) in subclasses
    
- ✅ Accessible via **reference of the parent type** but **only within the same package**
    
- ❌ Accessible via **reference of the parent type** in **different package** subclasses? **NO**

In a non-modular JVM, `public` means accessible anywhere in that JVM

In the module system `public` means accessible anywhere within the same module
- and, if the module exports the containing package, also accessible anywhere within another module that reads the declaring module
Note that modules prohibit *split packages*

##### Quick Question
<span style="color: #F0F8FF;">What accessibilities allow the code to compile?</span>
```Java
class P {/* accessibilities? */ int x = 99;}
class Q extends P {
	void show (P p) {
		System.out.println(p.x);
	}
}
```
Answer: public, protected, default

<span style="color: #F0F8FF;">What accessibilities allow the code to compile?</span>
``` Java
package p1;
class P {/*accessibility??*/ int x = 99;}

package p2;
class Q extends p1.P {
	void show(p1.P p) {System.out.println(p.x);}
}
```
Answer: public
