#### Omitting the `permits` clause
A syntax shortcut is provided to simplify development:
- If a `sealed` class and all its subtypes are declared in a single source file
	- The permits clause may be entirely omitted

#### Enums, Records and Sealed Hierarchies
Neither record nor enum types permit an extends clause
- However both can implement interfaces
A record type is implicitly final
- It can form a leaf node of a sealed hierarchy without being labeled as such

Enum types are not final as such, but any subtypes must be declared inside the enum's declaration
- This is sufficiently controlled to allow enums to be directly included as leaf nodes of sealed hierarchies
- The `final` keyword cannot be used

#### Recap
For elements of a sealed hierarchy
- The root node must be sealed
	- And generally has a permits clause
- Leaf nodes may be final, non-sealed, record, or enum types
- Intermediate nodes may be sealed (with permits) or non-sealed
- A non-sealed type must have a parent type that's sealed
- Abstract classes and interfaces cannot form leaf nodes
	- Except as non-sealed "placeholders"
- In JPMS, the entire hierarchy must be in a single module
	- Otherwise, the whole hierarchy must be in a single package

##### Quick Question
<span style="color: #F0F8FF;">What do you notice?</span>
```java
public sealed interface Parent permits ChildA, ChildB {}
final enum ChildA implements Parent {}
final record ChildB() implements Parent {}
```
Answer:
The `enum` cannot be marked `final`, so this fails
Without that, it would be valid.
Making the record `final` is redundant, but valid

<span style="color: #F0F8FF;">What do you notice?</span>
```Java
package sweet;
import nonsweet.Tomato;
public sealed class Fruit permits Orange, Tomato {}
final class Orange extends Fruit {}
-----------------------------
package nonsweet;
import sweet.Fruit;
public final class Tomato extends Fruit {}
```
Answer: This only works if run under JPMS

Given
```Java
package tasty;
public sealed class Fruit {}
final class Orange extends Fruit {}
```
and:
```Java
package tasty;
public final class Tomato extends Fruit {}
```
What do you notice?
Answer: Since both `Fruit` and `Tomato` are public, they must be in separate source files.
Because of that, the `permits` is mandatory

##### Question Deep Dive
<span style="color: #F0F8FF;">What is the result?</span>
```java
abstract class Y {
	int x = 100;
	Y(int v) {
		x = v;
		announce();
	}
	abstract void announce();
}

public class X extends Y {
	int x = 300;
	X() {super(200);}
	void announce() {System.out.println("x is " + x);}
	public static void main(String[] args) {
		new X();
	}
}
```
Answer:
- A) Compilation Fails
- B) `x is 300`
- C) `x is 200`
- D) `x is 100`
- E) `x is 0` (X)
