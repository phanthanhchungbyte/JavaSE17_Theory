#### Fields and Methods in Enum Types
Java's enum types are permitted permitted fields, methods, and nested types, just like regular classes
- Declare them following the semicolon-terminated value list
- They may be static or instance
#### The API of Enum Instances
The base type of all enum types is the class `java.lang.Enum`
All enum types provide these instance methods

| `name()`           | This method is final and returns the declared name of the enum value                                                          |
| ------------------ | ----------------------------------------------------------------------------------------------------------------------------- |
| `toString()`       | This non-final method returns the declared name of the enum value                                                             |
| `compareTo`        | Enum values implement `Comparable` and are ordered according to their source code declarations                                |
| `equals/ hashCode` | These are implemented according to normal conventions, but note that equivalence is best tested using `==` rather than equals |
| `ordinal()`        | This tells the zero-based index of the value in the declaration list                                                          |
#### Finding Enum Instances
Enum types can be obtained from their constants of from text
Given:
```java
enumm Suit {
	HEART, DIAMOND, CLUB, SPADE;
}
```
allows: `Suit s = Suit.DIAMOND;`
and/or: `Suit s = Suit.valueOf("HEART");`
and/or: `Suit s = Enum.valueOf(Suit.class, "HEART");`
The `valueOf` throw exceptions for unknown text

```java
enum Suit {
    HEARTS("coers"), DIAMONDS("carreaux"),
    CLUBS("trèfles"), SPADES("piques");
    private final String translation;
    Suit(String f) { translation = f; }
    public String getTranslation() { return this.translation; }
    public String toString() {
    StringBuilder res = new StringBuilder(this.name().toLowerCase());
    res.setCharAt(0, Character.toLowerCase(res.charAt(0)));
    return res.toString();
    }
}

public class TrySuit {
    public static void main(String[] args) {
    System.out.println(Suit.values()[0] + " : " + Suit.values()[0].getTranslation());
    }
}
```
Answer: `Hearts : coers`

##### Question Deep Dive
Given:
```Java
enum Constants { PI(3.14), E(2.72), AVOGADRO(6.02E23);
  private double value; }
```

Which are true?

A) 
if
```java
Constants(double d) {value = d;}
```
is added, compilation succeeds

B) 
if 
```java
private Constants(double d) {value = d;}
```
is added, compilation succeeds

C) 
if 
```java
public Constants(double d) {value = d;}
```
 is added, compilation succeeds
 
D)
adding 
```java
public String toString() {
    return super.toString() + " : " + value;
}
```
does not cause failure

E) 
adding 
```Java
public String presentation() {
    return "Constant " + super.toString() + " : " + value;
}
```
 does not cause failure

F) 
adding 
```Java
public String name() {return "Constant";} 
```
does not cause failure

Answer:
A) Constructor automatically private, takes a double arg, suitable and assign correctly (OK)
B) Constructor explicitly private, also true (OK)
C) public constructor not permitted for enum (NOT OK)
D) valid and would not cause failure
E) valid and would not cause failure
F) the name method in the underlying enum is a final method, can't override it
