#### Valid Lambda Contexts
When using a lambda expression, the compiler must be able to
determine unambiguously which functional interface is to be
implemented. This information comes from the context.

Four contexts support this
- Assignment to an l-value expression (e.g., variable)
- The actual parameter of a method invocation
	- but method overloads might cause ambiguity
- The value of a method's `return` statement
- A cast

#### The L-Value Context
Assignment does not have to be to a simple variable

Assuming
```java
interface PS { boolean test(String st); }
```

This is valid
```java
PS [] psa = { null };
psa[0] = s -> s.length() > 3;
```
and so is this
```java
PS [] psa = { s -> s.length() > 3 };
```
#### The Actual Parameter Context
Lambda expressions can be passed as method arguments

Assuming PS as before and:
```java
public static void announce (PS test, String s) {
	out.println(test.test(s) ? "Yes!" : "No.");
}
```
This is valid
```java
announce(s -> s.length() > 3, "Hello");
```

#### The Return Value Context
Lambda expressions can be returned from methods

Assuming PS as before, this is valid
```java
public static PS makeTest (int threshold) {
	return s -> s.length() > threshold;
}
```
(Notice threshold? We'll have more to say about that shortly.)

#### The Casting Context
Lambda expressions can be created in a cast context

Assuming PS as before, this is not permitted
```java
Object test = s -> s.length() > 3; // FAILS
```
However, this is valid
```java
Object test = (PS)s -> s.length() > 3;
```
Casts can have multiple bounds
```java
Object test = (PS & Serializable) s -> s.length() > 3;
```

#### Closures with Lambda Expressions
Lambda expressions can create a closure, exactly as local classes
- and any variables captured in this way must be marked final, or be effectively final

##### Quick Question
What happens?
```java
public JButton makeButton() {
	int[] count = {0};
	JButton jb = new JButton("Press Me");
	jb.addActionListener(e -> System.out.println("Click " + count [0] ++ + " times"));
	return jb;
}
```

Answer: This succeeds, the count variable is a reference, and its value is never modified.
The value in the array is modified, but that's valid.
