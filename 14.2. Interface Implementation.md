#### Implementing an Interface
When a concrete class that declares it implements an interface
- that class must provide concrete method implementations for all the abstract methods of that interface, including any resulting from interfaces that it extends
There is no such requirement if an *abstract* class declares that it implements an interface, nor if an interface declares that it extends an interface
- The resulting type is *still abstract*, and an abstract type may contain abstract methods. 

#### Unimplemented Abstract Methods
If an abstract class implements an interface, or if an interface extends another interface
- It's permissible to leave some abstract methods unimplemented
- in which case, those methods remain abstract and any concrete class that derives from this point must still provide an implementation
Note that the source of the derived type can simply ignore the methods that it leaves abstract, so their existence will not be visible simply by looking at that type

#### The @Override Annotation
The `@Override` annotation may be used
- when declaring an abstract method that exists in a parent type
- when a default method provides an implementation for an abstract, or a default, method in the parent

##### Quick Question
Which are valid?
```java
interface T {void doSomething(); void doSomethingElse();}
interface U extends V {default void doSomething();}
interface V extends U {void doSomething();}
class A implements T {}
abstract class B implements T {}
class C implements V {public void doSomethingElse() {}}
class D implements U {public void doSomethingElse() {}}
```
Answer: T , U, V, B and D
