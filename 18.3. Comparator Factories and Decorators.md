#### Function Factories / Decorators
If written to do so, an implementation of `Comparator` candelegate part of the decision to other `Comparable`(s) or `Comparator`(s)

Delegating in this way allows, for example, writing a method that takes a `Comparator` as an argument, and creates a new`Comparator` that reverses the ordering provided by the argument `Comparator`

This type of behavior might be called a function factory, a function decorator or a higher order function

#### Higher Order Functions of `Comparator`
The `Comparator` interface can create many implementations using the factory/decorator approach
- `naturalOrder`, `reverseOrder --` delegate to a `Comparable`
- comparing -- use a Function to extract a value, then order by natural, or Comparator, order of that value.
	- also variants extracting int, long, and double values.
- `nullsFirst(comparator)`, `nullsLast(comparator)`
- `reversed()`
- `comparing`
	- variants taking a function that extracts a `Comparable` field
	- variants taking a function that extracts a primitive
	- variants taking a function that extracts a value and delegating to another `Comparator` to order that value
- `thenComparing --` delegates to a second comparator if the first returns zero
	- variants as `comparing`
- `Collections` also provides `reverseOrder()` and `reverseOrder(comparator)`

##### Quick Question
Assuming the `Student` class has a string name (which might be null) and
an `int` grade field, with getter methods for both, what will this code
do to a `List<Student>` students?
```java
students.sort(
	Comparator.comparing((Student s) -> s.getName(),
	Comparator.nullsFirst(String.CASE_INSENSITIVE_ORDER)).thenComparingInt(s -> s. getGrade())
);
```
Answer: Sorts the list by ascending alphabetical order of name, ignoring case, and where two names are the same sub-orders by ascending grade. Null names are ordered ahead of non-null names
