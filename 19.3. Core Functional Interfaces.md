#### The Need for Functional Interfaces
A lambda may only be used in the context of a functional interface

So, a number of standard interfaces (over 40) exist in the core APIs to address the most common (but not all) scenarios

Generics reduces many requirements to the same form

But, generics cannot provide for varying "arity" (the number of arguments a method takes), nor void return values, nor primitives (without autoboxing, which is inefficient)

#### The `Function` Interface
To declare a lambda that takes a single reference-type argument, and returns a reference-type result, use the `Function` interface
```java
package java.util.function;

public interface Function<A, R> {
	R apply(A a);
	// static and default utility methods
}
```

Real example:
```java
import java.util.function.Function;

public class FunctionExample {
    public static void main(String[] args) {
        // Converts a String to its length
        Function<String, Integer> stringLength = s -> s.length();
        
        Integer length = stringLength.apply("Hello");
        System.out.println(length); // 5
    }
}
```

**What it does:**  
`Function` takes one reference-type argument and returns a reference-type result. The lambda implements `apply()`. Here, it maps a `String` to an `Integer`.
#### The `Consumer` Interface
To declare a lambda that takes a single reference-type argument, and has a void result, use the Consumer interface
```java
package java.util.function;

public interface Consumer<A> {
	void accept (A a);
	// static and default utility methods
}
```

Real example:
```java
import java.util.function.Consumer;

public class ConsumerExample {
    public static void main(String[] args) {
        Consumer<String> printer = message -> System.out.println("LOG: " + message);
        
        printer.accept("User logged in");
        // Output: LOG: User logged in
    }
}
```

**What it does:**  
`Consumer` takes one reference-type argument and returns `void`. Useful for side effects like printing, logging, or saving to a database.
#### The `Supplier` Interface
To declare a lambda that takes zero arguments, and returns a reference-type result, use the `Supplier` interface
```java
package java.util.function;

public interface Supplier<R> {
	R get();
}
```

Real example:
```java
import java.util.function.Supplier;

public class SupplierExample {
    public static void main(String[] args) {
        Supplier<Double> randomSupplier = () -> Math.random();
        
        Double value = randomSupplier.get();
        System.out.println(value); // e.g., 0.342567...
    }
}
```

**What it does:**  
`Supplier` takes no arguments and returns a reference-type result. Good for lazy generation of values.
#### Primitive Arguments and Returns
Primitive arguments and return types can be handled through autoboxing, however this is less efficient
- many variants of standard functional interfaces exist with prefixes indicating their arguments or return types are primitive
`ToInt`
`ToLong`
`ToDouble`

Return type is primitive 

`Int`
`Long`
`Double`

Argument type is primitive (except `XxxSupplier`)

Note that only `int`, `long`, and `double` are supported by the core API primitive variants
- Modern CPUs rarely compute in less than 32 bit words
	- smaller types might use less memory in arrays
- Java does not perform arithmetic smaller than `int` anyway

Method names in the versions of the interfaces that return primitive types are generally changed
```java
interface ToIntFunction<A> {
	int applyAsInt(A a);
}
```

Real example:
```java
import java.util.function.ToIntFunction;

public class PrimitiveFunctionExample {
    public static void main(String[] args) {
        // Returns primitive int, avoids autoboxing
        ToIntFunction<String> lengthAsInt = s -> s.length();
        
        int len = lengthAsInt.applyAsInt("Java");
        System.out.println(len); // 4
    }
}
```

**What it does:**  
Primitive variants like `ToIntFunction<T>` return `int`, `long`, or `double` directly, avoiding the overhead of autoboxing to `Integer`, `Long`, or `Double`.
#### Two Argument Forms
Most of these interfaces that exist in two-argument variations
- which generally have a prefix Bi in their name

For example
```java
package java.util.function;

public interface BiFunction<A, B, R> {
	R apply(A a, B b);
	// static and default utility methods
}
```
and
```java
public interface BiConsumer<A, B> {
	void apply(A a, B b);
}
```

Real example:
```java
import java.util.function.BiFunction;

public class BiFunctionExample {
    public static void main(String[] args) {
        BiFunction<String, String, String> concatenator = (a, b) -> a + " " + b;
        
        String result = concatenator.apply("Hello", "World");
        System.out.println(result); // Hello World
    }
}
```

**What it does:**  
`BiFunction` takes two arguments and returns one result. `BiConsumer` takes two arguments and returns `void`.
#### Boolean Returns
A lambda that returns a boolean result is directly supported
```java
package java.util.function;

public interface Predicate<A> {
	boolean test(A a);
	// static and default utility methods
}
```
and
```java
public interface BiPredicate<A, B> {
	boolean test (A a, B b);
	// static and default utility methods
}
```

Real example:
```java
import java.util.function.Predicate;

public class PredicateExample {
    public static void main(String[] args) {
        Predicate<String> isLongWord = word -> word.length() > 5;
        
        System.out.println(isLongWord.test("elephant")); // true
        System.out.println(isLongWord.test("cat"));      // false
    }
}
```

**What it does:**  
`Predicate` tests a value and returns `boolean`. Commonly used for filtering conditions.
#### The Operator Interfaces
Two interfaces exist to constrain argument and return types for functions
```java
package java.util. function;

public interface UnaryOperator<A> extends Function<A, A> {}
```
and
```java
public interface BinaryOperator<A> extends BiFunction<A, A, A> {}
```

Real example:
```java
import java.util.function.UnaryOperator;
import java.util.function.BinaryOperator;

public class OperatorExample {
    public static void main(String[] args) {
        UnaryOperator<String> shout = s -> s.toUpperCase() + "!";
        System.out.println(shout.apply("hello")); // HELLO!
        
        BinaryOperator<Integer> sum = (a, b) -> a + b;
        System.out.println(sum.apply(5, 3)); // 8
    }
}
```

**What it does:**
- `UnaryOperator` – same argument and return type (extends `Function<T,T>`).
- `BinaryOperator` – both arguments and return type are the same (extends `BiFunction<T,T,T>`).
#### More Functional Interfaces
Some more functional interfaces are spread around the core Java APIs including
- `java.lang.Runnable`
	- declares `void run();`
- `java.util.concurrent.Callable<V>`
	- declares `V call()` throws `Exception;`
- `java.util.Comparator<T>`
	- declares `int compare(T t1, T t2);`
	- and default and static utility methods

Real example:
```java
import java.util.concurrent.Callable;
import java.util.Comparator;

public class LegacyFunctionalExample {
    public static void main(String[] args) {
        // Runnable
        Runnable task = () -> System.out.println("Running in thread");
        task.run(); // Running in thread
        
        // Callable
        Callable<String> callable = () -> "Result from callable";
        try {
            String result = callable.call();
            System.out.println(result); // Result from callable
        } catch (Exception e) { }
        
        // Comparator
        Comparator<String> byLength = (a, b) -> Integer.compare(a.length(), b.length());
        System.out.println(byLength.compare("cat", "elephant")); // negative (-1 roughly)
    }
}
```

**What it does:**  
These pre-Java 8 interfaces are also functional interfaces. `Runnable` takes nothing and returns nothing, `Callable` returns a value and can throw an exception, `Comparator` compares two objects.
#### Utility Methods in Functional Interfaces
Many functional interfaces declare utility methods, either as static or default methods
- Many are factories
	- `Function Function.identity()`
	- `Comparator Comparator.comparing(Function keyExtractor)`
- Others combine operations
	- `Function Function.andThen(Function other)`
	- `Predicate Predicate.and(Predicate other)`

Real example:
``` java
import java.util.function.Function;
import java.util.function.Predicate;

public class UtilityMethodsExample {
    public static void main(String[] args) {
        // Function.identity() – returns the input itself
        Function<String, String> identity = Function.identity();
        System.out.println(identity.apply("same")); // same
        
        // Function.andThen() – chain operations
        Function<String, Integer> getLength = s -> s.length();
        Function<Integer, String> lengthToString = l -> "Length: " + l;
        Function<String, String> combined = getLength.andThen(lengthToString);
        System.out.println(combined.apply("Hello")); // Length: 5
        
        // Predicate.and() – combine conditions
        Predicate<String> longerThan3 = s -> s.length() > 3;
        Predicate<String> startsWithA = s -> s.startsWith("A");
        Predicate<String> combinedPredicate = longerThan3.and(startsWithA);
        
        System.out.println(combinedPredicate.test("Apple"));   // true (length>3 and starts with A)
        System.out.println(combinedPredicate.test("Hi"));      // false
    }
}
```

**What it does:**
- `Function.identity()` – returns a lambda that just returns its argument.
- `Function.andThen()` – applies the first function, then passes the result to the second.
- `Predicate.and()` – logically combines two predicates with AND.
These utility methods allow functional composition without writing extra boilerplate.
