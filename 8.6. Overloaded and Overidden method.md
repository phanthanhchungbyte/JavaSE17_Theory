,#### Method Overloading
The "full name" of a method is made up of
- The fully qualified classname
- The method's short name
- The type-sequence of the arguments
Two methods in different classes are always different methods, though they might have a special relationship
Two methods in the same class with the same name, but different argument type sequences are different methods, and *have no syntactic relationship* -- this is overloading.

Overloading methods cannot differ *solely* by their return types

Overloading can occur with a method from a parent class
- static or instance
- but again, the argument type sequence must differ

#### Non - differences for Overloading
Some argument types are not actually different
- Array forms: `[]` and `...`
- Generic variations `List<TypeA>` and `List<TypeB>`
Method modifiers don't count as differences either:
- `synchronized, strictfp, native,`
- `final`, and
- access control (`public` etc.)
There's no difference in the call site, so the compiler cannot distinguish the intended target

#### Resolving Overloaded Methods
When code invokes an overloaded method, the compiler must decide which of the alternatives to bind to. This decision is:
- entirely static (made by the compiler)
- entirely based on the types of the invocation's arguments
	- In particular, the return type is not considered
	- Consequently, if two methods differ only in return type they will not compile
- promotion, boxing, and varargs add complexity to this

Overload resolution follows an order:
1. A perfect match of argument type and count always wins
	- widening promotions may be applied
	- look for the "nearest/shortest" promotion
	- ambiguity causes compilation failure
2. Autoboxing/unboxing may be tried
	- will not change basic type
	- ambiguity causes compilation failure
3. Varargs may be tried
	- ambiguity causes compilation failure

##### Quick Question
Which of these:
```Java
public static void doStuff(int x, long y) {}
public static void doStuff(long x, long y) {}
public static void doStuff(int x, int... y) {}
```
is called by this
```Java
doStuff(99, 100);
```
Answer:
```Java
doStuff(int x, long y)
```

Which of these:
```Java
public static void doStuff(int x, long y) {}
public static void doStuff(long x, int y) {}
public static void doStuff(int x, int... y) {}
```
is called by this
```Java
doStuff(99, 100);
```
Answer: Compilation fails

