#### A Working Definition of Encapsulation
An encapsulated class protects itself against potentially invalid changes in its state that are made from the outside.

#### Implementing Encapsulation
In Java this generally means
- behaviors that result in newly created objects should always produce valid state
- methods that mutate data should validate the proposed change before proceeding
- mutable fields should be private
- internal mutable state should not be caller-provided
- internal mutable state should not be returned or otherwise made accessible to callers
- subclassing must be controlled

##### Example
```Java
package encapsulation;

public class Date {
	private int day;
	private int month;
	private int year;
	
	public Date(int day, int month, int year) {
		if (isValid(day,month,year)) {
			this.day = day;
			this.month = month;
			this.year = year;
		} else throw new IllegalArgumentException("Bad values for a date");
	}
	
	public static boolean isValid(int day, int month, int year) {
		return day >= 1 && day <= 31 && month >= 1 && month <= 12;
	}
	
	public void setDay(int day) {
		if (isValid(day, this.month, this.year)) {
			this.day = day;
		} else throw new IllegalArgumentException("Bad date values");
	}
	
	public void setMonth(int month) {
		this.month = month;
	}
	
	public void setYear(int year) {
		this.year = year;
	}
	
	public getDay() {
		return this.day;
	}
	
	public getMonth() {
		return this.month;
	}
	
	public getYear() {
		return this.year;
	}
}
```
