#### Fundamentals of Generics
Java's generics mechanism provides a flexible *compile-time* type checking mechanism
- Unlike most of Java's typing, the generic aspect of type does *not* necessarily persist to the running software
	- this is called *type erasure*
- This is necessary for backward compatibility
	- it also keeps the system light-weight
	- but it makes it possible to cheat the system and cause errors at runtime

#### Reviewing Generics Fundamentals
Generic type parameters are associated with an expression, not with a type

And, every expression has its own value for that type parameter

#### Assigning/Inferring Generic Types
Generic types/type parameters can be *assigned*:
`List<String> names;` assignment to a new variable
`new ArrayList<String>()` assignment to the expression created
`List.<String>of` assignment in a method invocation

Generic types can often be *inferred* from arguments and return type usage:
`new ArrayList<>(...)`
`List.of(...)`

#### Generics, instanceof and Casting
Because generic type parameters on types are erased during compilation `instanceof` cannot test them
this is valid:
```java
Object list = new ArrayList<String>();
System.out.println(list instanceof List);
```

but this is not:
```Java
System.out.println(list instanceof List<String>);
```

A cast to a generic type is permitted:
```Java
List<String> ls = (List<String>) list
```
but although this verifies that the object is a `List`, it does not (and *cannot*) verify the contents

##### Quick Question
What happens?
```Java
new ArrayList<>(List.of("")).add(LocalDate.now());
```
Answer: Compilation fails

What happens?
```Java
List l = new ArrayList<String>();
l.add(LocalDate.now());
List<String> ls = l;
String s = ls.get(0);
```
Answer: Execution fails, throwing a `ClassCastException`

What happens?
```Java
List<String> ls = new ArrayList<>();
if (ls instanceof ArrayList) {
	System.out.println("Yes");
} else {
	System.out.println("No");
}
```
Answer: Print `Yes`
