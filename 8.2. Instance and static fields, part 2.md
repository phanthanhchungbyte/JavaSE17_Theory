#### Resolving Unqualified Identifiers
An *unqualified* identifier is one that has no dot in front of it.
- Qualified: myObj,**theField**
- Unqualified; myObj
Unqualified identifiers will be searched for as:
- Method locals
- For each level of class hierarchy
	- If in an instance context, with the implied prefix `this`
	- with the implied class prefix

#### Multiple Potential Resolutions
Sometimes the resolution process has multiple possibilities
- At the same level of the type hierarchy
- At different levels of type hierarchy
At different levels, the "closer" is always default

- Disambiguate using
	- a type name prefix for static members
	- an instance reference prefix
	- a cast
	- `super` in an instance context (only one level!)

#### Comparing static and Instance
- Static features are associated with the "concept" as a whole
	- with the class, not to each instance
	- only one variable, shared from all instances
	- accessible with a class prefix
	- Static members can generally be prefixed with an instance reference (except interface static methods)
- Instance features are related to specific instances
	- must have an instance prefix (perhaps implicit `this`)

##### Quick Question
What happens?
```Java
class Stat {
	static int x = 99;
	
	public static void main(String[] args) {
		Stat s1 = new Stat();
		Stat s2 = new Stat();
		s1.x++; s2.x++; s2.x++;
		System.out.println(s1.x);
	}
}
```
Answer: Prints 102

#### Additional Field Rules
- `var` is not permitted for fields (only for method locals)
- Access to a class field can use an instance prefix
	- e.g.:aThing.aStaticField
	- But this is bad style
- Fields do not have any kind of dynamic binding
	- *Compile-time type* of prefix decides the field to be accessed
- Fields in interfaces are implicitly public, static and final
	- But are not restricted to constant expression initialization
- `super` cannot be used to access fields of interfaces
	- unless such a field has been incorporated into a parent class of this class.

```Java
package fieldaccess;

class Parent {
	public int x = 100;
}

public class InstanceField extends Parent {
	int x = 200;
	
	public void go() {
		System.out.println("InstanceField's version of x is " + x);
		System.out.println("Parent's version of x is " + super.x);
		Parent p = this;
		System.out.println("p.x gives: " + p.x);
	}
	
	public static void main(String[] args) {
		InstanceField ifld = new InstanceField();
		ifld.go();
	}
}

```

##### Quick Question
What happens?
```Java
class X {
	String name = "X";
}
class Y extends X {
	String name = "Y";
}

class Scratch {
	public static void main(String[] args) {
		X x = new Y();
		System.out.println(x.name + " : " + ((Y) x).name);
	}
}
```
