#### The Concept of Reduction
A binary operator takes two data items of a given type and creates one item of the same type
- The reduce operation applies this repeatedly to create a single result from an arbitrary number of items

This is the foundation of how the stream API produces a single result from many input items

#### Observations About Reduction

The simple view of reduction deserves some more consideration
- What happens if there are no data items?
	- "Add up" has a zero value
	- but for some operations there is no meaningful value
- What happens if the result type should be different from the input data type?
	- or if the intermediate steps of the calculation require something different, e.g., averaging?
	- or if we need a "final processing" step?

Average in progress must be "sum so far" and "count so far"

##### Quick Question
If there is "no meaningful value" for a reduction, how might this best be represented?

Answer: return an `Optional`

#### Observations About Reduction
More considerations about reduction
- What if the operations are not grouped left to right?
	- an associate operation will be unaffected
	- e.g., numeric addition and string concatenation
- What if the operations are not performed "in order"?
	- a commutative operation will be unaffected
	- e.g., numeric addition, but NOT string concatenation

If the binary operation of a reduction is not a pure function, many potential optimizations are unsafe.

#### The `reduce` Methods of Stream
The stream API provides reduce methods that correspond to these discussions
Where T is the stream data type, and U is the different result type
```java
Optional<T> reduce(BinaryOperator<T> acc)
I reduce(T identity, BinaryOperator<T> acc)
U reduce(U identity,
	BiFunction<U,? super T, U> accumulator,
	BinaryOperator<U> combiner)
```

Example 1:
```java
// Example: Find the longest string
List<String> names = Arrays.asList("Fred", "Jim", "Sheila", "Alexander");
Optional<String> longest = names.stream()
    .reduce((s1, s2) -> s1.length() > s2.length() ? s1 : s2);

longest.ifPresent(System.out::println); // Output: Alexander

// Empty stream example
Optional<Integer> emptySum = Stream.<Integer>empty()
    .reduce((a, b) -> a + b);
System.out.println(emptySum.isPresent()); // Output: false
```

Example 2:
```java
// Example: Sum of integers
List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);
int sum = numbers.stream()
    .reduce(0, (a, b) -> a + b);
System.out.println(sum); // Output: 15

// Empty stream returns identity
int emptySum = Stream.<Integer>empty()
    .reduce(0, (a, b) -> a + b);
System.out.println(emptySum); // Output: 0

// String concatenation
List<String> words = Arrays.asList("Hello", " ", "World");
String sentence = words.stream()
    .reduce("", (a, b) -> a + b);
System.out.println(sentence); // Output: Hello World
```

Example 3:
```java
// Example 1: Sum of string lengths (Stream<String> → Integer)
List<String> names = Arrays.asList("Fred", "Jim", "Sheila");
int totalLength = names.stream()
    .reduce(0,
        (acc, name) -> acc + name.length(),      // Accumulator
        (partial1, partial2) -> partial1 + partial2 // Combiner
    );
System.out.println(totalLength); // Output: 13 (4+3+6)

// Example 2: Collect into HashSet (Stream<String> → Set<String>)
List<String> words = Arrays.asList("hello", "world", "hello");
Set<String> unique = words.stream()
    .reduce(
        new HashSet<>(),                          // Identity
        (set, word) -> {                          // Accumulator
            set.add(word);
            return set;
        },
        (set1, set2) -> {                         // Combiner
            set1.addAll(set2);
            return set1;
        }
    );
System.out.println(unique); // Output: [hello, world]

// Example 3: Parallel processing with different types
List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);
String concatenated = numbers.parallelStream()
    .reduce(
        "",                                       // Identity (String)
        (acc, num) -> acc + num.toString(),       // Accumulator
        (str1, str2) -> str1 + str2               // Combiner
    );
System.out.println(concatenated); // Output: 12345
```

##### Quick Question
On a `Stream<String>` what is the return type of the single argument `reduce` method?
Answer: `Optional<String>`

One of the reduce methods takes three arguments. What are their types and purposes?

Answer:
The first argument is the zero / identity value of the result type.

The second argument is the `BiFunction` that accepts one item of the result type and one item of the stream type, and returns a new item of the result type.

The third argument is a `BinaryOperator` that accepts two items of the result type and creates a value that represents the combination of those two.
