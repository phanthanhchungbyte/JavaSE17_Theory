#### General Compilation Requirements
For any compilation process, this information will be needed
- which sources to compile
	- and where to find them
- which supporting binaries are needed
	- and where to find them
- where to put the output files

Note that this course will mainly focus on command line operations that are specific to the module system

#### Where to Put the Output
The compiler `javac` uses the `-d` flag to tell it where to put
output files. (Whether compiling modules or not)
- specify a directory following the `-d `flag:
	`javac -d ./classes`
- a directory hierarchy matching the packages will be created
- when compiling a module, the directory structure might include a directory reflecting the module name
	- this will happen if other directives indicate that a module is being compiled

Module binaries are conventionally placed on a path that includes the module name as a directory just below the root of the packages

#### Where to Find Source Files
Two mechanisms exist for finding source files
- List all the source files, with necessary path prefixes, on the compiler command line
- Use the `--module-source-path` option
	- this option only specifies where to find sources, other parts of the command line must specify which module or modules should be compiled
	- this option has no shorthand version
#### Which Source Files to Compile
If using the `--module-source-path` mechanism to locate sources
- which modules should be compiled is partly indicated using the `--module` option
	- with a list of the module names, separated with commas, but with no whitespace in the list
		`--module mod.one,mod.two`
	- `-m` is a shorthand for `--module`
- modules also specify, internally, what modules they need to support them
	- these specifications might add additional modules to those that will be compiled

#### What the Asterisk Means
The asterisk in a `--module-source-path` directive will be substituted with the module name to be compiled
- so, in our example
	`--module-source-path "./*/src"`
- causes the compiler to look for the source code in two directories
	`./test.module/src`
	`./module.two/src`
	
Note that the source-path cannot *start* with the `*`, and most OS shells require that it be escaped or quoted

#### Finding Compiled Modules
In any larger project, some binaries will be precompiled and simply need to be available to the compiler 
- list locations that contain modules using `--module-path`
- the option takes a delimited list of locations
	- in Unix-like systems, use colon
	- in Windows use semicolon
The binary modules might be represented as a directory containing the binary files in a tree structure, or as a zip/jar file with the name of the module containing the files in a directory structure.

```tree
dev/
- accounts.module/
	- src/
		- main/
			- java/
				- module-info.java
- credit.module/
	- src/
		- main/
			- java/
				- module-info.java
- modules/
```
Answer (all on one line):
`javac -d modules --module-source-path "./*/src/main/java" --module accounts.module,credit.module`
