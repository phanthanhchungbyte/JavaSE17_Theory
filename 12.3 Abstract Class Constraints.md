#### Abstract Type Basics
Abstract types are abstract classes and interfaces, they both
- permit the declaration of abstract methods
- must not be instantiated directly
Abstract classes
- cannot be final
- provide constructors to support initialization by their subclasses
	- perhaps the default constructor
	- private constructors only support nested subclasses

#### More Abstract Class Features
May define
- static and instance fields, with any accessibility
- concrete methods, both static and instance, with any accessibility
- nested classes, both static and instance, with any accessibility
- nested interfaces, which are implicitly static, with any accessibility
Abstract classes are still classes, and any subclass is still restricted to a single parent, whether abstract or not

#### Abstract methods
Abstract methods must
- be non-final
- be non-private
- be instance methods
- be terminated with a semicolon rather than a method body
Note that the keyword `default ` applies to methods *only in interfaces*
