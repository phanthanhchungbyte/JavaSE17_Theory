#### A Working Definition of Immutability
An immutable class prevents changes in its state, instead implementing operations that return a new object representing the consequences of an interaction

If objects are immutable, changes must be represented by creating a new object

#### Implementing Immutability
To implement immutability
- methods that represent state changes must return new objects
	- but those objects can share the unmodified fields from the original object
- caller-provided data must not be used unless it is primitive, immutable, or a copy is made
- internal mutable state must not be passed to the caller unless it is primitive, immutable, or a copy is given
- subclassing must also be controlled
