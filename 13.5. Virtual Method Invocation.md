#### Virtual Method Invocation
For non-private, non-final, instance methods
- the type of the *reference* determines the directly accessible methods at compilation time
- the type of the *object* referred to determines the behavior that results when invoked
Sometimes referred to as *late binding* or *dynamic method invocation*
- Also described as *sending a message* to the referred object

#### Revisiting `@Override`
Remember that the `@Override` annotation does not influence whether a method is, or is not, an override

It merely requests that the compiler issue an error if the method is not in fact an override

Methods implementing abstract methods, or replacing default methods, count as overrides for this annotation

#### Invoking the Overridden Method
In a class that overrides parent methods, the version of behavior that would have been exhibited by the parent class can be invoked directly by using the `super` prefix in place of an (implied or explicit) `this` prefix.
Syntax is of the form `super.doStuff();`
- This can be done anywhere the `this` value exists and refers to an instance of the subclass, not only the overriding method
- `super` only goes to "what you would have had", it cannot be chained

##### Quick Question
Making a change at `/* here */`, can this print `P.doStuff` and if so, how?
```java
class P {
	void doStuff() {out.println("P.doStuff");}
}
public class X extends P {
	void doStuff() {out.println("X.doStuff");}
	void doStuff2() {/* here */}
	public static void main (String[] args) {
		new X().doStuff2();
	}
}
```
Answer: insert `super.doStuff();`

What does this print, and if the `@Override` is uncommented, what changes?
```java
class P {
	void doStuff() {out.println("P.doStuff");}
}
public class X extends P {
	// @Override
	void doStuff() {out.println("X.doStuff");}
	public static void main(String[] args) {
		P p = new X();
		p.doStuff();
	}
}

```
Answer: `x.doStuff`.
- Nothing changes if `@Override` is uncommented
