#### Local Classes and their Restrictions
**Java allows classes to be declared in a block**
- e.g., in a method
- they're local to the block, so access control is not permitted
- they behave as instance or static depending whether the block has a `this` or not
	- static keyword is not permitted
Java 11 prohibits local enums and local interfaces

A local class might refer to a local value in the enclosing method
- This is permitted if, and only if, the value is either marked as `final` or is `effectively final`
	- Effectively final means it is definitely initialized and never reassigned
- This is called a *closure*
- Note that a final variable of reference type might refer to a mutable object, but cannot be reassigned to refer to a different object

##### What happens?
```Java
public Local getOne() {
	class Local {}
	return new Local();
}
```
Answer: compilation fails
Scope of Local is only inside the method, so it cannot be used for the return type

If Local implemented a well-known interface, that interface type could be used as the return type

```Java
public Serializable getOne() {
	public class Local implements Serializable {}
	return new Local();
}
```
Answer: compilation fails
Access control does not apply to local elements

#### Anonymous Classes
If a class is to be declared and instantiated only at the point of declaration, no name is required. This is an anonymous class
- Equivalent to "make this" while handling over instructions
- Multiple instances can be made, but only by re-executing that same line of code
- Anonymous classes often occur in methods, but can be used in any assignment, including field initialization
- Anonymous classes require a *single* generalized type (parent) to be indicated in the constructor syntax
	- Which can be class, abstract class, or interface type

##### What happens?
```java
public JButton makeButton() {
	JButton jb = new JButton("Press Me");
	jb.addActionListener(new ActionListener() {
		@Override
		public void actionPerformed(ActionEvent e) {
			System.out.println("Click!!");
		}
	});
	return jb;
}
```
Answer: Success

```Java
public JButton makeButton() {
	int count = 0;
	JButton jb = new JButton("Press Me");
	jb.addActionListener(new ActionListener() {
		@Override
		public void actionPerformed(ActionEvent e) {
			System.out.println("Click " + count++ + " times");
		}
	});
	return jb;
}
```
Answer: Compilation fails because the local variable count is modified

#### Footnote
Many uses of anonymous classes are superseded by lambda expressions, but anonymous classes can
- be subclasses of concrete or abstract classes
	- and pass arguments to a parent class constructor
- implement interfaces that have multiple abstract methods
- define and override any number of methods
- cont and fields and nested types
Instances of anonymous classes also have their own `this` reference
