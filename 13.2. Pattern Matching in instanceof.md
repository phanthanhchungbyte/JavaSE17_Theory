#### Patterns with `instanceof`
At Java 16 the `instanceof` operator gained expressiveness
- It has always been, and still is, an operator with a boolean result
- It can now declare and initialize a new variable of the tested type
	- The right side is now a *pattern*
- This changes a multi-step process of step-declare-cast-assign into a single expression
- Some generic expressiveness is added

```Java
public final class EqualsImplementation {  
  private int x, y;  
  
  public EqualsImplementation(int x, int y) {  
    this.x = x;  
    this.y = y;  
  }  
  
  @Override  
  public boolean equals(Object other) {  
    if (other instanceof EqualsImplementation otn) {  
      return this.x == otn.x && this.y == otn.y;  
    }  
    return false;  
  }  
}  
  
class TryTwoNums {  
  public static void main(String[] args) {  
    EqualsImplementation tn1 = new EqualsImplementation(1, 2);  
    EqualsImplementation tn2 = new EqualsImplementation(1, 2);  
    System.out.println(tn1.equals(tn2));  
  }  
}
```

#### Additional Notes
Some additional points are worth enumerating
- Pattern variables must not shadow existing variables
- Pattern variables are only initialized when the `instanceof` expression is `true`
- The pattern's type must be plausible for the type being tested
	- e.g., an `Integer` type cannot be tested for being a `CharSequence`
And, for any `instanceof`
- Test of `null` are always `false`

##### Quick Question
Given:
```Java
Number num = Double.valueOf(3.14);
if (/*insert code here*/) {
	out.println("num is an Integer with value " + i1);
}
```
What should be added at `/* insert code here */` to complete this code?
Answer: `num instanceof Integer i1`

Given:
```Java
Object o1 = "Hello";
out.println("String, longer than 3 chars? " + /*Insert code here*/);
```
What should be added at `/*Insert code here*/` to have the effect of printing `true` when `o1` refers to a `String` longer than 3 characters?
Answer: `o1 instanceof String st && st.length() > 3`
