#### Rules for Default Methods
A default method must not match the signature of any method of `java.lang.Object`

```Java
interface W {  
  default boolean equals (Object o) {return true;}  
    
  default void doStuff() {  
    System.out.println("");  
  }  
}  
class A implements W {  
  
}  
public class Resolve {  
  public static void main(String[] args) {  
    new A().doStuff();  
  }  
}
```

A default method provides a "fallback" implementation only; any implementation of the same signature in the class hierarchy will always be used in preference

```Java
interface W {  
  default boolean equals (String o) {return true;}  
  
  default void doStuff() {  
    System.out.println("default W.doStuff");  
  }  
}  
  
class B {  
  public void doStuff() {  
    System.out.println("B.doStuff");  
  }  
}  
  
class A extends B implements W {  
  
}  
  
public class Resolve {  
  public static void main(String[] args) {  
    new A().doStuff();  
  }  
}
```

#### Delegating to a Default Method
When a class implements a default method, it can delegate to that default method using the "qualified super" syntax
`InterfaceName.super.overriddenMethod(...)`

```java
class A extends B implements W {  
  public void doStuff() {  
    System.out.println("A.doStuff");  
    W.super.doStuff();  
    super.doStuff();  
  }  
}  
  
public class Resolve {  
  public static void main(String[] args) {  
    new A().doStuff();  
  }  
}
```

#### Rules for Default Methods
If a class implements a default method, the default is no longer visible to subclasses of that class.
```java
class B extends A {  
  public void doStuff() {  
    System.out.println("A.doStuff"); 
    W.super.doStuff(); // this line not allowed since class A has a concrete implementation,
							  // default method is essentially hidden 
    super.doStuff();  
  }  
}  
  
class A implements W {
	public void doStuff() {
		System.out.println("A.doStuff");
	}
}
```


If a class acquires *more than one* override-equivalent methods from interfaces, and *any of them* are default methods, that class itself must declare the method, possibly by inheriting it from a parent.

```java
interface IV {  
  default void doStuff() {};  
}  
interface IW {  
  void doStuff();  
}  
class A2 implements IV, IW {  
  @Override  
  public void doStuff() {  
    System.out.println("A2.doStuff");  
  }  
}  
  
public class Resolve2 {  
  public static void main(String[] args) {  
    new A2().doStuff();  
  }  
}
```

##### Quick Question
Does this work, if not, what can be done about it?
```java
interface T {default void doSomething() {} }
interface U {default void doSomething() {} }
class V implements T, U {}
```
Answer: Compilation of V fails (two default methods with same signature)
To fix, Implement `doSomething in class V`
Can refer to `T.super.doSomething()` and/or `U.super.doSomething()` in that implementation
Or make both default methods, and class V, abstract

##### Question Deep Dive
Given P and Q, which class are valid?
```java
interface P { default void doStuff() {}}  
interface Q { void doStuff() {}}  
  
abstract class A implements P, Q {  
  abstract void doStuff() {}  
}  
  
abstract class B implements P, Q {  
  public abstract void doStuff() {}  
}  
  
class C implements P {}  
class D implements P, Q {}  
class E implements P {  
  public void doStuff() {  
    super.doStuff();  
  }  
}
```
Answer: B, C
